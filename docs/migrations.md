# 配置迁移

配置顶层 `version` 是稳定兼容边界。当前只支持 `version: 1`。

升级规则：

1. 同一主配置版本内只增加带安全默认值的可选字段。
2. 删除或改变字段语义时发布新的配置版本和独立迁移命令。
3. 服务不会自动覆盖旧文件；迁移先输出新文件并通过完整校验。
4. Web 保存前保留备份，可通过 Settings 或 `/api/config/rollback` 回滚。
5. 新版本二进制必须继续读取上一个配置版本，至少保留一个稳定发行周期。

遇到 `unsupported config version` 时，应使用与文件版本匹配的旧二进制或项目提供的迁移命令，不能手工只修改版本号。
